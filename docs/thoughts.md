## 问题

1. 创建任务模板流程有问题，时间选择器有问题，不能打开
2. 所有的表单中时间相关字段都应该有默认值，开始时间就选择当前实现， 结束时间的话想一个优雅的方案
3. 很多对话框同时有 x 号和 取消等按钮，是不是可以优化一下，移除 x 按钮
4. 还有任务依赖关系图有两个对话框的样子
上面的问题都应该分析所有模块是否存在同类问题并处理


1. 关键结果的 weight 权重属性的范围应该是 1-5
2. 很多表单的国际化配置没有实现，比如goal的创建表单，我用的中文，还是用英文。

1. goal 模块还有 task 模块的这类卡片 存在 高低不一的问题， 想一个优雅的解决一下。我建议是标题、描述等内容都设定固定高度，两行或者几行，多余省略。
2. 日程模块的重点应该是日视图、周视图、月视图的日程展示规划。用来展示 task 或者 goal 或者 reminder 之类的日程安排，调度控制台应该是类似开发者工具吧
3. 仓储模块（知识库）的页面需要全面重构优化，不应该叫 知识库，直接就是 仓库 就行了， 然后，仓库的界面应该是类似 obsidian 的，左边是文件树（需要创建文件树组件）还有文件树里面的文件夹就固定为 笔记（md） 图片（jpg等）视频（mp4） 音频。。 等，就按照类型区分 不让用户创建文件夹，强迫用户使用 tags 元数据来分类、筛选资源。也不需要新建仓库了，每个人就一个仓库。 左边的文件树应该有 文件列表、搜索（标签、文件名搜索）、书签 三个模式，右边就是编辑（md笔记）、阅读查看区域。模仿 obsidian。 应该是支持批量导入资源（同时支持给资源批量添加标签）。
